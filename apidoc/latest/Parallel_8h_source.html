107 // A task scheduler is created/terminated automatically by TBB since
108 // version 2.2. It is recommended by Intel not to instantiate any task
109 // scheduler manually. However, in order to support the -threads option
110 // which can be used to limit the number of threads, a global task scheduler
111 // instance is created and the -threads argument passed on to its initialize
112 // method by ParseParallelOption. There should be no task scheduler created/
113 // terminated in any of the IRTK libraries functions and classes.
114 MIRTK_Common_EXPORT extern std::unique_ptr<task_scheduler_init> tbb_scheduler;
